NAMING CONVENTIONS:
-------------------

1. STAGING MODELS: stg_{source}__{entity}.sql
   - Example: stg_shopify__orders.sql
   - Double underscore separates source from entity
   - Always plural entity names

2. MART MODELS: {type}_{entity}.sql
   - dim_customers.sql (dimension = descriptive data)
   - fct_orders.sql (fact = measurable events/transactions)

3. YAML FILES: _{folder}__sources.yml or _{folder}__models.yml
   - Leading underscore keeps them at top of folder
   - Makes them easy to find
