PHASE 2: WHAT YOU'LL LEARN NOW
================================================================================

LESSON 1: dbt Layer Architecture
--------------------------------
File: 01_dbt_layer_architecture.txt

Why organize models into layers? You'll learn:
- Staging layer (you've done this!)
- Intermediate layer (business logic)  
- Marts layer (final analytics tables)
- How data flows between layers


LESSON 2: Dimensional Modeling
------------------------------
File: 02_dimensional_modeling.txt

The star schema pattern:
- What are FACT tables? (events, transactions)
- What are DIMENSION tables? (entities, attributes)
- Why this matters for analytics
- How it applies to your Shopify data


LESSON 3: Building Intermediate Models
--------------------------------------
File: 03_intermediate_models.txt

Reusable building blocks:
- When to create intermediate models
- Joining staging tables
- Business logic transformations
- Real example with your data


LESSON 4: Building Fact Tables
------------------------------
File: 04_fact_tables.txt

Your first mart:
- fct_orders (grain: one row per order)
- Key metrics: revenue, quantities, discounts
- Joining dimensions
- Implementation walkthrough


LESSON 5: Building Dimension Tables
-----------------------------------
File: 05_dimension_tables.txt

Entity reference tables:
- dim_customers
- dim_products
- Slowly Changing Dimensions (SCD)
- Implementation walkthrough


LESSON 6: The ref() Function Deep Dive
--------------------------------------
File: 06_ref_function.txt

Model dependencies:
- How dbt builds the DAG
- Proper use of ref()
- Avoiding circular dependencies
- Debugging dependency issues


LESSON 7: Advanced Testing
--------------------------
File: 07_advanced_testing.txt

Beyond unique and not_null:
- Relationship tests
- Accepted values
- Custom SQL tests
- dbt-expectations package
